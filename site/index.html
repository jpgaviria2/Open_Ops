The merchant reality

Small merchants are running modern businesses on a pile of disconnected tools.

Independent operators need the same digital leverage as big chains, but their tools rarely talk to each other.

  • POS data lives in one dashboard.
  • Ordering and loyalty live somewhere else.
  • Staff checklists are paper, spreadsheets, or Notion pages.
  • Inventory, waste, and discrepancies are reported manually.
  • Customer communication is scattered across social, SMS, DMs, and email.
  • AI tools exist, but they do not understand the actual business.

Hosted merchant onboarding

What a merchant provides — while Open Ops runs the stack.

Credentials & accounts

  • No merchant GitHub required for hosted plans — Open Ops owns repos, CI/CD, hosting, and cloud runtime.
  • Merchant provides DNS delegation/custom-domain records and approval for TLS/app association files.
  • Square production app: access token, location ID, catalog, orders, payments, inventory, KDS.
  • BTCPay Server choice: Open Ops hosted store/node or merchant-owned URL, store ID, API key, webhook secret.
  • Breez/Spark API key and LNURL domain if enabled; Nostr keys/relays/NIP-05; OpenClaw bot identity.
  • Apple Developer + App Store Connect API key; Google Play account + signing/upload key if native apps ship under merchant accounts.

Merchant assets

  • Legal/display/app names, tagline, support/privacy/terms URLs, app store descriptions and keywords.
  • Logo SVG/PNG, app icon source, PWA icons, Android adaptive icon, brand colors and fonts.
  • Storefront, product, team, station-standard, procedure, screenshot, and feature-graphic imagery.
  • Square menu/catalog: categories, items, modifiers, taxes, photos, pickup rules, KDS naming.

Operating inputs

  • Staff roster, roles, manager list, approved pubkeys, and escalation rules.
  • Opening/closing/inventory checklists, procedures, recipes, training cards.
  • Inventory items, units, par levels, vendors, waste/discrepancy categories, forecast thresholds.
  • Rewards policy: sats %, caps, expiry, refund handling, education copy, liquidity budget.
